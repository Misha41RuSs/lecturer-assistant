## Роли в системе

Система разделяет пользователей на две основные роли:
- **Лектор**: управляет процессом лекции через веб-панель управления (Frontend React-приложение).
- **Студент**: участвует в лекции через свой привычный мессенджер с помощью Telegram-бота.

---

## Руководство для Лектора (Web-интерфейс)

В качестве лектора вся работа происходит через единый веб-интерфейс.

### Начало работы и создание лекции
1. Откройте веб-приложение в браузере (по умолчанию панель управления доступна по адресу развёрнутого Frontend-приложения).
2. Для планирования новой лекции создайте её, указав **название** (которое будет использоваться студентами для подключения).
3. При необходимости установите **пароль** (Access Type) для ограничения доступа к лекции. Лекция изначально создается в статусе `DRAFT` (черновик).

### Управление презентацией и контентом
1. Вы можете загрузить файл презентации в формате **PDF** или **PPTX**.
2. Система автоматически обработает файл и преобразует его в последовательность слайдов.
3. При необходимости вы можете изменить порядок следования слайдов до начала лекции.

### Проведение лекции
1. Для начала лекции переведите её в статус `ACTIVE` (кнопка запуска).
2. Сообщите студентам название лекции (и пароль, если он задан) и ссылку на Telegram-бота.
3. Во время лекции вы управляете показом: переключение на следующий слайд автоматически отправляет актуальное изображение слайда всем подключенным в данный момент студентам в Telegram.
4. По завершении лекции, переведите её в статус `FINISHED`.

### Работа с аудиторией
- **Мониторинг студентов**: Лектор видит список всех подключенных студентов в реальном времени с их деанонимизированными данными (профилями Telegram).
- **Исключение (Kick)**: Если на лекцию зашел посторонний пользователь или нарушитель, лектор имеет возможность исключить (kick) конкретного студента из текущей сессии лекции.
- **Вопросы от студентов**: Студенты могут задавать вопросы через бота. Они появляются в специальном разделе панели лектора. Лектор может ответить на вопрос **публично** (ответ придет всем студентам) или **приватно** (ответ придет только автору вопроса).
- **Рассылка сообщений**: Доступна функция отправки текстовых сообщений или дополнительных изображений сразу всем студентам.

### Тестирование и опросы
1. **Создание**: Лектор может создать тесты (с правильными ответами) или опросы (без правильных ответов). Поддерживаются вопросы с множественным выбором и открытым ответом.
2. **Импорт/Экспорт**: Поддерживается импорт готовых тестов из файлов формата **GIFT**, а также экспорт тестов в этот формат.
3. **Запуск**: Тест можно запустить в любой момент лекции. При запуске тест автоматически рассылается подключенным студентам.
4. **Индивидуальные тесты**: Есть возможность выдавать (назначать) тесты конкретным пользователям (студентам) через интерфейс управления аудиторией.
5. **Проверка**: Закрытые вопросы проверяются автоматически, открытые вопросы могут быть оценены лектором (выставление балла).

### Аналитика
После завершения лекции доступен дашборд с аналитикой. В нем можно просмотреть:
- Хронологию подключений студентов.
- Тайминги смены слайдов.
- Итоговый отчет по успеваемости и вовлеченности аудитории.

## Руководство для Студента (Telegram-бот)

Для студентов не требуется установка дополнительных приложений, всё взаимодействие происходит в Telegram-боте ассистента.

### Подключение к лекции
1. Найдите Telegram-бота «Цифровой ассистент лектора» (ссылку или @username предоставляет лектор).
2. Для подключения к лекции отправьте команду:
   ```text
   /join <название_лекции>
   ```
3. Если лектор установил пароль, используйте формат:
   ```text
   /join <название_лекции> <пароль>
   ```
4. При успешном подключении бот пришлет подтверждение, и вы начнете получать актуальные слайды. *Ваш профиль Telegram будет использоваться для идентификации вас преподавателем.*

### Взаимодействие во время лекции
- **Просмотр слайдов**: Как только лектор переключает слайд, бот автоматически присылает вам картинку с новым слайдом.
- **Задать вопрос**: Если вам что-то непонятно, вы можете в любой момент задать вопрос лектору. Для этого используйте команду:
  ```text
  /question <ваш текст вопроса>
  ```
  Лектор ответит вам лично или отправит ответ на ваш вопрос всей группе.
- **Прохождение тестов**: Если лектор запускает тест или опрос, бот пришлет вам вопросы. Следуйте инструкциям бота, чтобы выбрать правильный вариант ответа или ввести текст (для открытых вопросов).
- **Завершение**: Когда лектор завершает лекцию, бот уведомит вас об окончании трансляции.
