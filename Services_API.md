# Описание микросервисов системы «Цифровой ассистент лектора»

Архитектура состоит из 4 основных микросервисов, взаимодействующих через API Gateway. Три сервиса имеют изолированные базы данных (Database-per-service), один (Трансляция) выступает в роли оркестратора.

---

## 1. Сервис управления учебными материалами (Content Management Service)
*Отвечает за хранение, выдачу и версионирование учебного контента. Инкапсулирует БД «Слайды»*.

### Модель данных
* **Slide (Слайд):** * `id` (UUID) — уникальный идентификатор слайда.
  * `title` (String) — название слайда.
  * `content` (JSON/BLOB) — содержимое (текст, изображения).
  * `version` (Integer) — текущая версия.
  * `created_at` / `updated_at` (DateTime) — даты создания и обновления.
  * `author_id` (UUID) — идентификатор автора.
* **SlideVersion (Версия слайда):** * `id`, `slide_id` (UUID) — идентификаторы версии и ссылки на слайд.
  * `version` (Integer) — номер версии.
  * `content` (JSON/BLOB) — содержимое на момент версии.
  * `created_at`, `updated_at` (DateTime).
  * `change_description` (String) — описание изменений.
* **SlideSequence (Последовательность слайдов):** * `id` (UUID), `name` (String).
  * `slides` (Array<UUID>) — список ID слайдов по порядку.
  * `created_at`, `updated_at` (DateTime).
* **CloudStorageLink (Связь с облаком):** * `id`, `slide_id` (UUID).
  * `storage_type` (String) — тип хранилища (Google Drive, Moodle).
  * `storage_url` (String) — URL файла в облаке.
  * `version` (Integer), `uploaded_at` (DateTime).

### API (Основные функции)
* **Получение текущей версии слайда:** `GET /slides/{slide_id}` — Возвращает текущую версию (поле content) .
* **Загрузка последовательности слайдов:** `POST /slide-sequences` — Тело: `{ "name": "...", "slides": [...] }` .
* **Запись изменений слайда:** `PUT /slides/{slide_id}` — Создает новую версию и обновляет текущую .
* **Версионирование слайдов:** `GET /slides/{slide_id}/versions` — Возвращает список всех версий слайда с их содержимым и историей изменений .
* **Интеграция с облачными хранилищами:** `POST /slides/{slide_id}/upload-to-cloud` — Загружает версию в облако и сохраняет ссылку .

## 2. Сервис тестирования (Quiz Service)
*Отвечает за опросы, тесты и сбор обратной связи от слушателей. Инкапсулирует БД «Тесты»* .

### Модель данных
* **Quiz (Тест/Опрос):** `id` (UUID), `lecture_id` (UUID), `title` (String), `created_at` (DateTime).
* **SlideQuestion (Вопрос к слайду):** `id` (UUID), `slide_id` (UUID), `user_id` (UUID), `text` (String).
* **UserResponse (Ответ слушателя):** `id` (UUID), `question_id` (UUID), `user_id` (UUID), `answer` (String).
* **SlideRating (Оценка слайда/материала):** `id` (UUID), `slide_id` (UUID), `user_id` (UUID), `score` (Integer).

### API (Основные функции)
* **Создание теста/задания:** `POST /quizzes` — Распределение тестов для аудитории.
* **Обработка ответа слушателя:** `POST /quizzes/{quiz_id}/responses` — Сбор ответов для последующего анализа.
* **Задать вопрос к материалу:** `POST /slides/{slide_id}/questions` — Интеграция с сервисом вопросов.
* **Оценка материала:** `POST /slides/{slide_id}/ratings` — Сохраняет оценку от слушателя.

## 3. Сервис аналитики (Analytics Service)
*Сбор телеметрии, логов и построение графиков активности. Инкапсулирует БД «Активности»*.

### Модель данных
* **SlideRequest (Запрос слайда):** `id` (UUID), `slide_id` (UUID), `user_id` (UUID), `timestamp` (DateTime).
* **ActivityLog (Лог активности):** `id` (UUID), `lecture_id` (UUID), `action_type` (String), `timestamp` (DateTime).
* **LectureSummary (Отчет):** `id` (UUID), `lecture_id` (UUID), `report_data` (JSON).

### API (Основные функции)
* **Сбор данных (события):** `POST /analytics/events` — Принимает данные о старте лекции и запросах слайдов от сервиса трансляции .
* **Обработка данных (Агрегация):** `GET /analytics/lectures/{lecture_id}/aggregations` — Анализирует количество запросов и выявляет проблемные слайды (много вопросов или нет запросов) .
* **Визуализация (Дашборд лектора):** `GET /analytics/lectures/{lecture_id}/dashboard` — Отдает данные для построения графиков активности слушателей в панели лектора.
* **Генерация отчета:** `GET /analytics/lectures/{lecture_id}/report` — Формирует итоговый отчет с рекомендациями после лекции.

---

## 4. Сервис трансляции (Broadcast Service)
*Оркестратор и движок реального времени. **Stateless** (не имеет собственной БД).*

### Модель данных (В оперативной памяти)
* **LectureSession:** `lecture_id` (UUID), `current_slide_id` (UUID), `status` (String).

### API (Основные функции)
* **Старт трансляции:** `POST /broadcast/start` — Инициирует лекцию и отправляет событие о старте в Сервис аналитики.
* **Демонстрация и изменение слайда:** `POST /broadcast/slides/{slide_id}/update` — Принимает отметки лектора и вызывает `PUT /slides/{slide_id}` в Сервисе учебных материалов для записи изменений.
* **Завершение трансляции:** `POST /broadcast/end` — Фиксирует конец лекции, запускает сбор финальной аналитики и отправку отчетов.

## Глобальные связи между сущностями (Relations)
Поскольку в системе применяется паттерн Database-per-service (базы данных микросервисов физически изолированы друг от друга), использование классических внешних ключей (Foreign Keys) на уровне СУБД невозможно. Для обеспечения целостности данных применяется механизм "мягких связей" (Soft Links): микросервисы обмениваются уникальными идентификаторами (UUID) на уровне бизнес-логики.

Для обеспечения целостности данных между изолированными микросервисами используются следующие логические связи по ID:
* **Лекция (Lecture)** связана со **Слайдом (Slide)** через `lecture_id`.
* **Слайд (Slide)** связан с **Запросом слайда (SlideRequest)**, **Оценкой слайда (SlideRating)** и **Вопросом к слайду (SlideQuestion)** через `slide_id`.
* **Пользователь (User)** связан с **Запросом слайда (SlideRequest)**, **Оценкой слайда (SlideRating)** и **Вопросом к слайду (SlideQuestion)** через `user_id`.
